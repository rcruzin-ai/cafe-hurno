# Milestone 1: Cafe Hurno MVP — Complete

**Date:** 2026-03-06

## What Was Built

A mobile-first coffee ordering web app for Cafe Hurno, publicly deployed and free to run for 20-100 users.

## Features Delivered

| Feature | Details |
|---------|---------|
| Menu display | 4 drinks, hot/cold toggle, P70 flat price |
| Cart | Zustand store, persisted to localStorage |
| Order placement | Google login required, inserts to Supabase |
| Feedback | Star rating + comment after order placed |
| Order tracking | Realtime status updates via Supabase Realtime |
| Admin dashboard | All orders, filter by status, advance status |
| QR code generator | Links to /menu, embeds logo, downloadable PNG |
| Google OAuth | Via Supabase Auth |
| Mobile-first | Bottom nav, 375px+ optimized |

## Key Technical Decisions

- **No backend server** — Supabase handles DB, auth, and realtime
- **RLS with is_admin() helper** — avoids infinite recursion when checking admin role from within policies on the profiles table
- **force-dynamic on client pages** — prevents Next.js from prerendering pages that use Supabase browser client
- **Flat pricing** — simplified data model, no price variants per size

## To Add More Admins (Future)

Run in Supabase SQL Editor after user has signed in:
```sql
update public.profiles set role = 'admin' where email = 'email@example.com';
```

Or update the trigger in schema.sql to include the email in the `in (...)` list.
